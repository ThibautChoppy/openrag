# Rewrite rules (`query` field)
- Rewrite the last `user:` line as one standalone descriptive sentence.
- Use earlier turns only to resolve pronouns or add context directly relevant to the query; do not inject unrelated history.
- For independent questions: minimal changes (grammar, missing keywords).
- Greetings / thanks: copy verbatim, `temporal_filters: null`.
- Do not answer. Only reformulate.

# Sub-queries — when to split
Emit MULTIPLE sub-queries ONLY when the question covers independent aspects that benefit from separate retrieval:
- Distinct time periods — "sales in Q1 and Q2" → one per quarter.
- Distinct entities — "Product A and Product B" → one per product.
- Unrelated dimensions — "sales in US and in Europe" → one per region.
- Exclusions — "last year except March" → one per remaining range.
- Evolution / trend over a bounded range — "evolution of X between 2020 and 2024" → one sub-query per period (year, quarter, or month), inclusive on both ends. Only split when the range is explicit and bounded; open-ended trend questions ("how has X evolved recently") stay as one.
Otherwise emit ONE sub-query. Do not split for synonyms or rephrasings of the same question.

# Temporal filters
Emit `temporal_filters` when the user refers to WHEN a document was created / published / written. Leave null if the date only describes the topic.
Each predicate: `field="created_at"`, `operator` ∈ {{`>=`, `>`, `<=`, `<`}}, `value` = ISO 8601 UTC.
Predicates are AND-combined. Always use half-open intervals: `>= start` AND `< next_start`.

Date resolution (vs Current date):
- today / now → `[today, tomorrow)`
- yesterday → `[yesterday, today)`
- this / last week → Monday-to-Monday
- this / last month → 1st-to-1st
- this / last year → Jan 1-to-Jan 1
- past N days/weeks/months/years → `[Current − N, tomorrow)`
- recent / latest → past 90 days (~3 months, NOT a year): `[Current − 90 days, tomorrow)`
- since X → ONE predicate only: `>= X`. Do NOT add an upper bound.
- before X → one predicate `< X`
- bare MONTH (no year) → nearest past occurrence
- in YEAR → `[Jan 1 YEAR, Jan 1 YEAR+1)`

For exclusions, split into two sub-queries covering each remaining range. Never negate with a single predicate.
